1. Objetivo de la Semana

Esta semana estuvo dedicada al análisis en profundidad del proceso de remodelación que ejecuta R.E Amueblamiento de Espacios S.A.S. El objetivo fue documentar de forma sistemática las fases y actividades que componen cada proyecto, establecer sus dependencias y construir el modelo conceptual de datos que servirá como base para el diseño del sistema de monitoreo.

2. Modelo del Proceso de Remodelación

Tras el levantamiento de información con el supervisor Ricardo Espinosa, se estableció que cada proyecto de remodelación sigue un proceso estandarizado compuesto por exactamente dos fases secuenciales, con un conjunto ordenado de actividades que deben ejecutarse en un orden definido.

3. Modelo Conceptual de Datos

A partir del análisis del proceso de remodelación, se identificaron las cinco entidades principales que debe gestionar el sistema:

EntidadDescripciónRelación
ProyectoCada remodelación de apartamento con sus datos básicosTiene hasta 2 Fases
FaseObra Blanca o Amueblamiento, asociada a un proyectoContiene Actividades
ActividadTarea específica con orden, estado y fecha estimadaGenera Alertas
UsuarioPersona que accede al sistema con un rol definidoGestiona el sistema
AlertaNotificación SMS generada cuando una actividad se retrasaVinculada a Actividad
