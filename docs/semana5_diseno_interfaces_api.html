1. Objetivo de la Semana

Esta semana se diseñaron las interfaces de usuario y se especificó la API REST completa del sistema Planmejora. Se definieron los wireframes de las 4 pantallas principales, el flujo de navegación entre ellas y la estructura de todos los endpoints del backend, incluyendo métodos HTTP, rutas, parámetros y formatos de respuesta.

Este diseño constituye el contrato entre el frontend y el backend, garantizando que la implementación de ambas capas pueda realizarse de manera coherente e independiente en las semanas siguientes.

4. Diseño de la API REST

4.1 Convenciones Generales

ConvenciónDetalle
Formato de datosJSON en todas las peticiones y respuestas (Content-Type: application/json)
AutenticaciónBearer token en el header Authorization: Bearer {token}
Métodos HTTPGET (consultar), POST (crear), PUT (actualizar completo), PATCH (actualizar parcial), DELETE (eliminar)
Respuesta exitosa{"success": true, "data": {...}}
Respuesta de error{"success": false, "error": "mensaje de error"}
Códigos HTTP200 OK, 201 Created, 400 Bad Request, 401 Unauthorized, 403 Forbidden, 404 Not Found, 500 Internal Server Error

4.2 Endpoints de la API

Proyectos
MétodoEndpointDescripción
GET/api/proyectos.phpListar todos los proyectos
GET/api/proyectos.php?id={id}Obtener proyecto por ID
POST/api/proyectos.phpCrear nuevo proyecto
PUT/api/proyectos.php?id={id}Actualizar proyecto completo
PATCH/api/proyectos.php?id={id}Cambiar estado del proyecto
DELETE/api/proyectos.php?id={id}Eliminar proyecto
Fases
MétodoEndpointDescripción
GET/api/fases.php?proyecto_id={id}Listar fases de un proyecto
POST/api/fases.phpCrear nueva fase
PUT/api/fases.php?id={id}Actualizar datos de una fase
Actividades
MétodoEndpointDescripción
GET/api/actividades.php?fase_id={id}Listar actividades de una fase
POST/api/actividades.phpCrear nueva actividad
PATCH/api/actividades.php?id={id}Cambiar estado de una actividad
Usuarios y Alertas
MétodoEndpointDescripción
GET/api/usuarios.phpListar todos los usuarios del sistema
POST/api/usuarios.phpCrear nuevo usuario
GET/api/alertas.phpConsultar historial de alertas enviadas
POST/api/alertas.phpRegistrar una nueva alerta
