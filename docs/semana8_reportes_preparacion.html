1. Objetivo de la Semana

Con esta semana se cierra la fase de diseño del sistema de monitoreo web. Los objetivos principales fueron diseñar el módulo de reportes, definir el plan detallado de implementación para las próximas semanas y preparar el entorno de desarrollo. Al finalizar esta semana el proyecto cuenta con todo el diseño completo y está listo para iniciar la codificación.

2.4. Mecanismo de exportación

El reporte se generará como una página HTML optimizada para impresión. El usuario podrá exportarla a PDF usando la función de impresión del navegador (Ctrl+P → Guardar como PDF). Este enfoque no requiere librerías adicionales ni procesamiento en el servidor.

6. Próximos Pasos — Semana 9

La semana 9 marcará el inicio de la implementación del sistema:

  • Crear el archivo de conexión a Supabase (backend/config/database.php).
  • Implementar el endpoint de proyectos (backend/api/proyectos.php) con operaciones CRUD completas.
  • Implementar el endpoint de fases y actividades.
  • Probar cada endpoint con herramientas de prueba HTTP (Postman o Thunder Client).
  • Iniciar la creación del archivo database/schema.sql con el script completo de tablas.
